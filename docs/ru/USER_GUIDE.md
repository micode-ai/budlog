# BudLog — Руководство пользователя

Как пользоваться BudLog в роли **прораба** (Telegram-бот), **клиента** (веб-отчёт) и
**администратора** (панель). По нему же можно проверить каждую функцию от начала до конца.

> English: [../en/USER_GUIDE.md](../en/USER_GUIDE.md) · Техническая: [TECHNICAL.md](TECHNICAL.md)

Бот отвечает на языке пользователя (PL/RU/UA/EN/DE/ES/FR/BE/NL). Примеры ниже на русском.

---

## Что такое BudLog

BudLog — это **голосовой журнал стройплощадки**. Вместо форм и таблиц прораб в конце дня
просто **наговаривает в Telegram** — «залили фундамент, ушло 40 мешков цемента, завтра
электрик» — и BudLog превращает это в чистую запись по датам: **выполненные работы,
использованные материалы и фото с площадки**. Клиент бригады открывает одну ссылку и видит,
что именно происходило на объекте, день за днём, и может скачать PDF-отчёт — **без установки
приложений и без логина**.

Решает три повседневные проблемы небольших бригад:
- **Журнал никто не ведёт.** Записать голосом (или быстрым фото) — пара секунд, поэтому это
  реально делается: ни бумаг, ни приложения, которое надо осваивать.
- **Споры с клиентом.** Датированный журнал с фото + PDF-«акт» сложно оспорить.
- **Клиент ничего не видит.** Вместо звонков он получает живую ссылку-отчёт «только для чтения».

Всё **многоязычное** (бот отвечает на языке говорящего) и работает с телефона прямо на объекте —
нужны только Telegram и браузер.

## Для кого это

| Роль | Кто это | Что делает |
|---|---|---|
| **Прораб / бригадир** | Человек на площадке | Наговаривает/шлёт фото в Telegram-бот, фиксируя работы, материалы и планы. Главный пользователь. |
| **Стройкомпания / подрядчик** | Бригада или небольшая фирма | Владеет аккаунтом и площадками; решает, кто может вести журнал (прораб = editor, остальные = viewer). |
| **Клиент / заказчик** | Тот, кто платит за работы | Открывает ссылку-отчёт — журнал, фото и PDF, только для чтения. Без аккаунта и приложения. |
| **Админ / поддержка** | Тот, кто эксплуатирует BudLog | Через админ-панель смотрит справки (пользователи, компании, площадки). |

**Кому подходит лучше всего:** небольшим строительным бригадам и частным подрядчикам (ремонт,
дома, отделка), которым нужен журнал без лишних усилий и убедительный отчёт для клиента —
особенно бригадам, работающим в PL / RU / UA. Это **не** смета, не бухгалтерия и не
проджект-менеджмент — BudLog делает одно дело хорошо: ежедневный журнал площадки и отчёт для
клиента.

## A. Прораб — Telegram-бот

### 1. Привязать аккаунт
Бот — **@BudlogBot** (или ваш бот от @BotFather).
1. Получите одноразовый код привязки. В деве — через API:
   ```bash
   TOKEN=$(curl -s -X POST localhost:3000/api/v1/auth/login -H 'Content-Type: application/json' \
     -d '{"email":"alice@test.com","password":"TestPass123"}' | grep -o '"accessToken":"[^"]*"' | sed 's/.*:"//;s/"//')
   ACC=$(curl -s localhost:3000/api/v1/accounts -H "Authorization: Bearer $TOKEN" | grep -o '"id":"[^"]*"' | head -1 | sed 's/.*:"//;s/"//')
   curl -s -X POST localhost:3000/api/v1/users/me/telegram-link-code -H "Authorization: Bearer $TOKEN" -H "X-Account-Id: $ACC"
   ```
2. В Telegram: `/start`, затем `/link <КОД>` → «Аккаунт привязан».

### 2. Создать площадку
```
/newsite Dom Kowalski
```
→ «🏗 Площадка создана и выбрана». Активная площадка — туда идут новые записи.

### 3. Записать работы голосом (основная функция)
Отправьте **голосовое**: *«залили фундамент, ушло 40 мешков цемента, завтра электрик»*.
Бот распознаёт речь, понимает смысл и показывает **карточку подтверждения**:
```
Подтвердите:
📍 Dom Kowalski
🔨 Работы: залили фундамент
📦 Материалы: цемент ×40 bags
📅 План: электрик
[ ✅ Подтвердить ]  [ ❌ Отмена ]
```
Нажмите **✅** → «Записано в Dom Kowalski». Ничего не сохраняется до подтверждения.

То же самое можно просто **написать текстом** — работает одинаково.

### 4. Прикрепить фото
Отправьте **фото** с подписью (например, «армирование 2 этаж»). Бот сохранит его к активной
площадке → «📷 Фото сохранено». Если в подписи есть работы/материалы — предложит записать и их.

### 5. Просмотр и управление
| Команда | Что делает |
|---|---|
| `/today` | журнал за сегодня по активной площадке (работы + материалы + фото) |
| `/site` | список площадок, переключение активной (кнопки) |
| `/newsite <название>` | создать площадку |
| `/report` | публичная ссылка на отчёт для клиента (+ ссылка на PDF) |
| `/account` | переключить компанию/аккаунт |
| `/help` | полный список команд |

## B. Клиент — веб-отчёт

Прораб выполняет `/report` и присылает ссылку вида `https://budlog.pl/r/<токен>`
(в деве: `http://localhost:3001/r/<токен>`). Логин не нужен.

Вы видите чистый журнал по датам:
- Название площадки, ваше имя, адрес, период.
- За каждый день: выполненные **Работы**, использованные **Материалы** (название × количество), **Фото** (тап — увеличение, стрелки / Esc).
- Кнопка **Скачать PDF** → печатный отчёт («акт»).
- Переключатель языка PL/RU/UA/EN (справа сверху).

Ссылку прораб может отозвать в любой момент (тогда страница покажет «Отчёт недоступен»).

## C. Администратор — панель поддержки

По адресу `https://budlog.pl/admin` (в деве: `http://localhost:3001/admin`).
1. Войдите под email из `ADMIN_EMAILS` (в деве: `alice@test.com` / `TestPass123`).
2. Дашборд показывает **карточки-счётчики** (пользователи, аккаунты, площадки, работы, материалы, фото, ссылки на отчёты)
   и три вкладки: **Users**, **Accounts**, **Sites** (с числом работ/материалов/фото по каждой площадке).

Только чтение — для справок поддержки, не для редактирования.

## Чек-лист проверки

- [ ] `/start` + `/link <код>` → привязано
- [ ] `/newsite Dom Kowalski` → площадка создана и активна
- [ ] Голос «залили фундамент, 40 мешков цемента, завтра электрик» → карточка → ✅ → записано
- [ ] Текстовое сообщение → тот же флоу подтверждения
- [ ] Фото с подписью → «Фото сохранено»
- [ ] `/today` → показывает вышеперечисленное хронологически
- [ ] `/site` → переключение активной площадки
- [ ] `/report` → открыть ссылку → отчёт рендерится → **Скачать PDF** работает → сменить язык
- [ ] `/admin` → вход → дашборд со счётчиками и таблицами
- [ ] Прораб второго аккаунта не видит площадки первого (изоляция данных)
